# 雀魂麻将识别助手 — 产品规格说明书

> 版本: 1.0 | 日期: 2026-05-02 | 阶段: 第一阶段

## 1. 产品概述

雀魂麻将识别助手，实时截取 Steam 版雀魂游戏画面，准确识别当前对局状态并输出结构化数据。

### 1.1 目标用户

雀魂麻将玩家，需要实时获取对局结构化信息用于辅助决策。

### 1.2 核心价值

将游戏画面中的人眼可读信息转化为机器可读的结构化数据，为后续辅助功能（牌效计算、防守建议等）提供数据基础。

## 2. 识别目标

### 2.1 手牌

| 属性 | 说明 |
|------|------|
| 位置 | 屏幕底部居中 |
| 数量 | 13-14张（摸牌后14张） |
| 副露影响 | 每组副露减少3张手牌，杠不减少 |
| 特殊状态 | 新摸的牌与手牌之间有间隔；可操作的牌有发光效果 |
| 牌面类型 | 万子1-9、筒子1-9、索子1-9、字牌7种（东南西北白发中），共37类 |
| 赤宝牌 | 五万、五筒、五索以红色"5"显示，需作为独立类别识别 |

### 2.2 宝牌指示器

| 属性 | 说明 |
|------|------|
| 位置 | 屏幕上方中央偏左 |
| 数量 | 初始1张，每次杠后增加1张，最多5张 |
| 含义 | 指示牌的**下一张**为实际宝牌（如指示牌为3万，则4万为宝牌） |
| 赤宝牌 | 红五自带宝牌属性，无需指示器 |

### 2.3 局次信息

| 属性 | 说明 |
|------|------|
| 位置 | 屏幕上方 |
| 内容 | 场风（东/南）、局数（一~四）、本场数（数字） |
| 示例 | "东一局 0 本场"、"南三局 2 本场" |
| 供托棒 | 立直后产生的供托棒数量 |

### 2.4 各家分数

| 属性 | 说明 |
|------|------|
| 位置 | 各玩家名字旁（四角） |
| 格式 | 带逗号的数字（如 25,000） |
| 起始分 | 四人战 25,000，三人战 35,000 |
| 变化动画 | 分数变化时有 +/- 数字弹出动画 |
| 排名 | 伴随显示当前排名（1st-4th） |

### 2.5 各家牌河（舍牌池）

| 属性 | 说明 |
|------|------|
| 位置 | 屏幕中央四个方向（上下左右） |
| 排列 | 按行列排列，最近的弃牌在最内侧 |
| 数量 | 不固定，随对局进行增加 |
| 视觉变形 | 对面和侧面玩家的牌有轻微3D透视变形 |
| 特殊标记 | 立直宣言牌横放；有人和了时，荣和的那张牌可能被标记 |

### 2.6 副露（吃/碰/杠）

| 属性 | 说明 |
|------|------|
| 位置 | 玩家手牌区域左侧（自己），对手在牌河附近 |
| 视觉特征 | 被叫的牌旋转90度横放，指示来源玩家 |
| 横放规则 | 来自右家→最右牌横放；来自对家→中间牌横放；来自左家→最左牌横放 |
| 杠特殊 | 暗杠中间两张朝下；明杠横放牌在最上；加杠在碰的基础上加第4张 |

### 2.7 动作提示

| 属性 | 说明 |
|------|------|
| 位置 | 屏幕底部中央偏右，手牌下方 |
| 出现时机 | 仅当对应操作可用时动态弹出 |
| 按钮列表 | 吃、碰、杠、荣和、自摸、立直、过（跳过） |
| 视觉干扰 | 可操作按钮有发光/闪烁/脉冲动画效果 |

### 2.8 出牌倒计时

| 属性 | 说明 |
|------|------|
| 位置 | 手牌区域附近 |
| 形式 | 数字倒计时 + 环形/条形进度指示 |
| 出现条件 | 仅轮到自己时可见 |
| 时间设置 | 3+5s、5+10s、5+20s、60+0s、300+0s |

### 3.1 牌面编码规则

| 编码 | 牌面 |
|------|------|
| `1m`-`9m` | 万子 1-9 |
| `1p`-`9p` | 筒子 1-9 |
| `1s`-`9s` | 索子 1-9 |
| `1z`-`4z` | 东南西北 |
| `5z`-`7z` | 白发中 |
| `5mr`/`5pr`/`5sr` | 赤宝牌（红五万/筒/索） |

## 4. 准确率目标

| 识别目标 | 准确率目标 | 测试条件 |
|----------|-----------|---------|
| 手牌 | ≥ 99% | 无动画干扰的静止帧 |
| 宝牌指示器 | ≥ 99% | 同上 |
| 各家分数 | ≥ 99% | 分数动画结束后 |
| 局次信息 | ≥ 99% | — |
| 各家牌河 | ≥ 97% | 全部牌河 |
| 副露 | ≥ 95% | 含横放牌 |
| 动作提示 | ≥ 99% | 静止状态 |
| 出牌倒计时 | ≥ 99% | — |

## 5. 约束条件

- 全部使用本地模型推理，不依赖云端 API
- 跨平台支持：macOS、Windows
- 目标运行环境：普通办公电脑（无独显亦可运行）
- **仅支持四人麻将对局**（三人麻将作为后续扩展）
- **仅支持简体中文 UI**（游戏语言设置需为简体中文）

## 6. 用户前置条件

使用本工具前，用户需在雀魂游戏设置中：

- 关闭角色立绘（避免遮挡牌河和手牌区域）
- 关闭对话气泡
- 游戏语言设置为简体中文
- 建议使用默认牌面主题（非特殊皮肤）
