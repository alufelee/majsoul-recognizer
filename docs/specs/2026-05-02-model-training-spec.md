# 雀魂麻将识别助手 — 模型训练与数据方案

> 版本: 1.0 | 日期: 2026-05-02

## 1. 训练数据合成方案

### 1.1 素材采集

**牌面素材**（从游戏中截取）：

| 素材类型 | 数量 | 说明 |
|----------|------|------|
| 正面牌面 | 37 种 × 2 状态（正常+发光）= 74 张 | 万子1-9、筒子1-9、索子1-9、字牌7种、赤宝牌3种 |
| 横置牌面 | 37 种 × 1 = 37 张 | 副露中的横放牌 |
| 牌背 | 3-5 种 | 不同皮肤的牌背样式 |
| 牌面 alpha 通道 | 37+37 张 | 去背景用的透明度信息 |

**背景素材**：

| 素材类型 | 数量 | 说明 |
|----------|------|------|
| 桌面背景 | 5+ 种 | 不同牌桌主题的背景截图 |
| 牌河背景 | 5+ 种 | 各主题的空牌河区域 |
| 手牌区背景 | 5+ 种 | 各主题的空手牌区域 |

**素材采集方式**：
- 在游戏中进入自由对局/友人战
- 截取各种状态下的清晰牌面
- 使用图像编辑工具裁剪出单张牌面并去除背景

### 1.2 合成流程

```
┌──────────┐   ┌──────────────┐   ┌──────────────┐   ┌──────────┐
│ 牌面素材  │   │  随机组合放置  │   │  YOLO标注生成  │   │ 数据增强  │
│ 背景素材  │──►│  模拟真实场景  │──►│  自动生成     │──►│  泛化处理 │
│ Alpha通道 │   │              │   │  .txt标注文件  │   │          │
└──────────┘   └──────────────┘   └──────────────┘   └──────────┘
```

**合成步骤**：

1. **选择背景**：随机选择一个桌面主题背景
2. **放置手牌**：在手牌区域位置，按间距排列 13-14 张随机牌
   - 模拟新摸牌的间隔：第 14 张牌与第 13 张之间加大间距
3. **放置牌河**：在各牌河区域位置，按行列排列随机数量的牌
   - 模拟透视变形：对面和侧面牌河添加轻微透视变换
4. **放置副露**：在副露区域放置 0-4 组副露
   - 模拟横放牌：对被叫的牌旋转 90 度
5. **放置宝牌**：在宝牌区域放置 1-5 张随机牌
6. **自动生成 YOLO 格式标注**（我们知道放的是哪张牌和位置）

### 1.3 数据增强

每张合成图额外生成 5-10 个变体：

| 增强方式 | 参数范围 | 目的 |
|----------|---------|------|
| 亮度调整 | ±15% | 适应不同显示器设置 |
| 对比度调整 | ±10% | 同上 |
| 高斯模糊 | kernel 1-3 | 模拟截图质量波动 |
| JPEG 压缩 | quality 70-95 | 模拟压缩伪影 |
| 随机噪声 | σ = 5-15 | 泛化能力 |
| 透视变换 | ±5度 | 模拟窗口缩放误差 |
| 颜色抖动 | 色相±5, 饱和度±10% | 适应不同主题配色 |
| 牌面发光叠加 | 随机 10-30% 牌添加发光效果 | 模拟可操作牌状态 |

### 1.4 数据量规划

| 数据集 | 数量 | 说明 |
|--------|------|------|
| 合成训练集 | 15,000 张 | 覆盖各种牌型组合和桌面主题 |
| 合成验证集 | 2,000 张 | 训练过程中验证 |
| 真实验证集 | 200 张 | 从实际游戏截图手动标注 |
| 真实测试集 | 100 张 | 最终精度评估，不参与训练 |

**真实数据集标注**：
- 从实际游戏中截取 300 张对局画面
- 使用 LabelImg 或 CVAT 手动标注所有牌面位置和类别
- 按 2:1 比例分为验证集和测试集

### 2.3 训练流程

```
阶段 1: 合成数据预训练
├── 使用 15,000 张合成图训练
├── 训练 100 epochs
├── 在真实验证集上评估
└── 如果 mAP@0.5 < 0.95，增加合成数据量

阶段 2: 真实数据微调
├── 使用 200 张真实标注数据微调
├── 降低学习率（lr0=0.001）
├── 训练 50 epochs
└── 在真实测试集上评估

阶段 3: 导出部署
├── 导出为 ONNX 格式
├── 量化为 FP16（减少模型体积）
└── 性能基准测试
```

### 2.4 精度评估标准

| 指标 | 目标值 | 说明 |
|------|--------|------|
| mAP@0.5 | ≥ 0.95 | 整体检测精度 |
| mAP@0.5:0.95 | ≥ 0.80 | 严格定位精度 |
| 手牌区 recall | ≥ 0.99 | 手牌不能漏检 |
| 牌河区 recall | ≥ 0.97 | 牌河允许少量遗漏 |
| 赤宝牌 precision | ≥ 0.98 | 赤宝牌不能误判 |
| 推理速度 (ONNX CPU) | ≤ 50ms | 单帧推理延迟 |

## 3. PaddleOCR 配置

### 3.1 模型选择

使用 PaddleOCR 的超轻量模型（PP-OCRv4）：

| 组件 | 模型 | 大小 |
|------|------|------|
| 文本检测 | ch_PP-OCRv4_det | ~4.7MB |
| 文本识别 | ch_PP-OCRv4_rec | ~10.5MB |

### 3.2 分区域 OCR 配置

```python
# 分数区域配置
score_ocr_config = {
    "det": True,
    "rec": True,
    "rec_char_dict": "0123456789,",  # 只识别数字和逗号
    "det_limit_side_len": 320,       # 小区域，降低计算量

# 局次区域配置

    "det": True,

    "det_limit_side_len": 320,

    "rec": True,
    "rec_char_dict": "0123456789",

```

### 3.3 OCR 后处理

```python
def parse_score(raw: str) -> int:
    """解析分数: '25,000' → 25000"""
    cleaned = raw.replace(",", "").strip()
    value = int(cleaned)
    assert 0 <= value <= 200000, f"score out of range: {value}"
    return value

def parse_round(raw: str) -> dict:
    """解析局次: '东一局' → {wind: '东', number: 1}"""
    wind_map = {"东": "东", "南": "南", "西": "西", "北": "北"}
    num_map = {"一": 1, "二": 2, "三": 3, "四": 4}
    wind = None
    number = None
    for ch in raw:
        if ch in wind_map:
            wind = ch
        if ch in num_map:
            number = num_map[ch]
    return {"wind": wind, "number": number}
```

## 4. 部署方案

### 4.1 ONNX 导出

```bash
# YOLOv8 导出
yolo export model=best.pt format=onnx imgsz=640 simplify=True

# PaddleOCR 导出（使用 paddle2onnx）
paddle2onnx --model_dir inference/det --model_filename det.pdmodel \
    --params_filename det.pdiparams --save_file det.onnx
```

### 4.2 ONNX Runtime 推理

```python
import onnxruntime as ort

# 优先使用 GPU（如有），否则 CPU
providers = (
    ['CUDAExecutionProvider', 'CPUExecutionProvider']
    if 'CUDAExecutionProvider' in ort.get_available_providers()
    else ['CPUExecutionProvider']
)

session = ort.InferenceSession("best.onnx", providers=providers)
```

### 4.3 性能预算

| 阶段 | 耗时预算 | 说明 |
|------|---------|------|
| 截图 | ≤ 10ms | mss 截屏 |
| 缩放+区域分割 | ≤ 5ms | OpenCV resize |
| YOLOv8 推理 | ≤ 50ms | ONNX Runtime CPU |
| OCR 推理 | ≤ 50ms | PaddleOCR 轻量版 |
| 模板匹配 | ≤ 10ms | OpenCV matchTemplate |
| 规则校验 | ≤ 5ms | 纯逻辑 |
| **总计** | **≤ 130ms** | **满足 200ms 目标** |

## 5. 持续优化

- 运行过程中自动保存低置信度检测的截图

- 定期用新数据微调模型

### 5.2 A/B 测试

- 新模型先与旧模型并行运行，对比结果
- 仅在新模型精度优于旧模型时替换
